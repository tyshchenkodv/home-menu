# Specifications

This directory contains immutable historical specifications and implementation
plans for substantial Home Menu changes.

## Workflow

1. Create `docs/specifications/<slug>/SPEC.md` after discovery, brainstorming,
   and `grill-me`.
2. Obtain explicit user approval for the specification.
3. Create `PLAN.md` beside the approved specification.
4. Obtain explicit user approval for the plan.
5. Implement, verify, and update the current project documentation.
6. Mark the entry as implemented in this index.

Approved specifications and plans are historical records. Do not edit them
after approval. If requirements materially change, create a new specification
and link it with `Supersedes`.
